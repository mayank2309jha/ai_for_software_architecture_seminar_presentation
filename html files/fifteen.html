1. Decoupled Ingestion: Orders are instantly written to Kafka, separating request handling from processing.
2. Parallel Workers (No Locks): Each worker handles a partition → no shared state → no locking → high throughput.
3. Replayable State: Workers rebuild state from Kafka logs → durability without database bottlenecks.
